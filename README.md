# AcetAngle

AcetAngle — это система для анализа рентгеновских снимков с использованием искусственного интеллекта. Система состоит из трех основных компонентов: серверной части (`server.py`), сервера для обработки запросов к модели (`inferenceServer.py`) и клиентского приложения (`client.py`).

## Обзор системы

### Компоненты

1. **server.py**
   - FastAPI-сервис для аутентификации пользователей и проксирования сообщений к внешнему AI-сервису.
   - Сохраняет историю чатов в MongoDB.
   - Обеспечивает регистрацию, аутентификацию и обновление данных пользователей.
   - Проксирует сообщения к AI-сервису и сохраняет историю чатов.

2. **inferenceServer.py**
   - HTTP-сервер, проксирующий запросы к LMStudio для анализа рентгеновских снимков.
   - Поддерживает два типа запросов:
     - `call_type=0`: Двухшаговый анализ снимка с получением контрольных точек, линий и углов, а также диагноза.
     - `call_type=1`: Простой чат-бот без строгой схемы.
   - Рисует разметку на изображении для отладки и второго шага анализа.

3. **client.py**
   - Клиентское приложение для взаимодействия с сервером.
   - Предоставляет интерфейс для загрузки изображений, анализа и просмотра результатов.
   - Поддерживает функции входа, регистрации и выхода из системы.

## Установка и запуск

### Требования

- Python 3.8+
- FastAPI
- Uvicorn
- Pydantic
- PyMongo
- Requests
- Pillow
- LMStudio (для работы inferenceServer.py)

### Установка зависимостей

```bash
pip install fastapi uvicorn pydantic pymongo requests pillow python-dotenv

**Требуется** установленный MongoDB и LMStudio

Запуск сервера:
uvicorn server:app --reload --port 8000

Запуск сервера инференса:
python inferenceServer.py
   
Запуск клиентского приложения:
python client.py

### Конфигурация

Для работы системы необходимо настроить переменные окружения в файле `.env`:

Например 
(желательно устанавливать ipv4)
```env
# .env
SERVER_URL=http://localhost:8000
AI_URL=http://127.0.0.1:25555
AI_API_KEY=1234567890
MONGO_DB=mongodb://localhost:27017

# Конфигурация
LMSTUDIO_URL = "http://127.0.0.1:1234/v1/chat/completions"   # адрес LMStudio
MODEL_NAME = "gpt-oss-20b"                                   # имя модели в LMStudio
EXPECTED_API_KEY = "1234567890"                              # замените на свой ключ (или None для отключения проверки)
```

### Замеченные недостатки
Очень долго пытается подключится к серверу инференса по ipv6

**inferenceServer.py**
Принимает json:
**Форма запроса к серверу инференса**
```
{
  	'api_key': 'api_key',                // ключ
  	'request_id': '1835',                // id запрашивающей сессии
    'call_type': '0'                     // тип запроса (0 первичный анализ, 1 вторичный (режим простого чатбота))
  	'question': 'Обнаружь патологию, опиши причины', // Вопрос нейросети (промт)
    'context': 'assistance: Окак\nuser: ага'
    'filename': filename,                // Имя файла снимка 
    'image': encoded_string              // Изображение в бинарной последовательности 
}
```

Первичный анализ состоит из двух этапов
1. Разметка
При первом проходе нейросеть проводит поиск точек
Парсер автоматически проводит линии и генерирует изображение идущее во второй этап
2. Поиск патологий
